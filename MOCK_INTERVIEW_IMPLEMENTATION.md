# 🎙️ Mock Interview Implementation Guide

This document explains the technical architecture and implementation details of the **AI-Powered Mock Interview** feature within the Drona AI Recruitment Engine.

---

## 🏗️ 1. High-Level Architecture

The system follows a three-tier architecture:
1.  **Frontend (React)**: Manages the Recruiter Dashboard and the Candidate Voice Room.
2.  **Backend (Flask)**: Orchestrates prompt generation, database persistence, and evaluation logic.
3.  **AI Layer**: 
    *   **Vapi**: Real-time Voice-to-Voice AI.
    *   **Gemini/Claude**: Dynamic prompt generation and post-interview evaluation.

---

## 🗄️ 2. Database Schema (`database.py`)

Two primary tables manage the interview lifecycle:

### `mock_interviews` (Campaigns)
Defines the job role and parameters.
- `id`: Primary Key.
- `title`: Job Title (e.g., "Senior Backend Engineer").
- `description`: The Job Description (used for AI context).
- `duration_minutes`: Interview time limit.

### `interview_sessions` (Candidate Entries)
Stores individual candidate attempts.
- `id`: Unique Session ID.
- `interview_id`: Foreign Key to `mock_interviews`.
- `candidate_name`: Name of the candidate.
- `resume_text`: Extracted text from the uploaded PDF.
- `transcript`: JSON blob containing the full conversation.
- `overall_score`: Generated by AI after the interview (0-100).
- `feedback`: Detailed AI feedback.

---

## 🔌 3. Backend Endpoints (`app.py`)

### `POST /api/interviews/config`
**Purpose**: Prepares the AI for a specific candidate.
1.  Receives `interview_id`, `candidate_name`, and `resume.pdf`.
2.  Extracts text from the PDF.
3.  **Prompt Generation**: Uses the `INTERVIEW_PROMPT_GENERATOR` to combine the JD and Resume into a custom system prompt:
    *   *"You are Drona AI, interviewing [Name] for [Role]. Use their resume [Text] and JD [Text] to ask deep technical questions..."*
4.  Returns the `system_prompt` and `session_id`.

### `POST /api/interviews/session/<id>/finish`
**Purpose**: Evaluates the candidate after the call ends.
1.  Receives the `transcript` from the frontend.
2.  Sends the `transcript` + `JD` + `Resume` to the `INTERVIEW_EVALUATOR` prompt.
3.  The LLM returns a JSON object with scores for technical skills, communication, and an overall rating.
4.  Updates the database with the results.

---

## 💻 4. Frontend Components

### `InterviewDashboard.jsx` (Recruiter View)
- **Matrix Grid**: Displays active campaigns as physical folders.
- **Analytics**: Uses a **Frequency Polygon** SVG chart to show candidate trends.
- **Actions**: Generates application links and launches test modes.

### `VoicePortal.jsx` & `VoiceRoom.jsx` (Candidate View)
- **Vapi Integration**: Initializes the Vapi Web SDK using the `publicKey`.
- **Real-time UI**:
    - Animated voice waves to indicate AI/Candidate speaking.
    - Countdown timer based on the campaign's duration.
    - Automatic cleanup and evaluation triggers when the call ends.

---

## 🧠 5. AI Prompt Engineering

The "Secret Sauce" lies in the `prompts.py` file:

1.  **Generator Prompt**: Instructs the AI to be a "strict but fair" technical interviewer, focusing on gaps between the resume and JD.
2.  **Evaluator Prompt**: A structured rubric that ensures consistency in scoring. It forces the LLM to output valid JSON for:
    - `overall_score`
    - `technical_rating`
    - `communication_rating`
    - `feedback`

## 🚀 6. Implementation Checklist

To replicate this in your project:
1.  [ ] **Install Vapi SDK**: `npm install @vapi-ai/web`.
2.  [ ] **Add PyMuPDF**: `pip install pymupdf` for resume text extraction.
3.  [ ] **Configure API Keys**: Set `GEMINI_API_KEY` and `VAPI_PRIVATE_KEY` in your `.env`.
4.  [ ] **Initialize DB**: Run the SQL schema to create the interview tables.
5.  [ ] **Set up Flask Routes**: Implement the config and finish endpoints.
6.  [ ] **Build the Voice Room**: Create a React component that calls `vapi.start(config)`.
